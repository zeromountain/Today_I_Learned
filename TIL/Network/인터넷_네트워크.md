# IP(인터넷 프로토콜)
- 인터넷 상에서 사용하는 실제 주소와 같은 역할
- 패킷: 택배의 송장 역할, 데이터를 전송하기 위한 규칙

![패킷](https://media.vlpt.us/images/syleemk/post/cd49e92c-66ad-4608-ace1-c57c4af36813/image.png)

- 클라이언트 패킷 전달: 패킷을 인터넷망의 노드를 통해서 목적지까지 전송

![클라이언트](https://media.vlpt.us/images/syleemk/post/196abc16-c217-46c4-b344-634a695f2514/image.png)
- 서버 패킷 전달: 서버 -> 클라이언트 목적지까지 노드 경로

![서버](https://media.vlpt.us/images/syleemk/post/d8c24ad0-85e8-447b-a479-b235f08b242a/image.png)

# TCP, UDP
- 인터넷 프로토콜 스택의 4계층

![4계층](https://media.vlpt.us/images/syleemk/post/8284fb54-f742-4e2c-8284-dc8503b0707b/image.png)

- 프로토콜 계층

![프로토콜계층](https://media.vlpt.us/images/syleemk/post/039ec647-355f-45c1-86fc-9c7bfa827481/image.png)
>1. 프로그램이 Hello, world! 메시지 생성
>2. SOCKET 라이브러리를 통해 OS계층에 메시지 전달
>3. TCP는 Hello, world 메시지에 TCP 정보를 씌운다.
>4. TCP 밑의 IP 계층으로 메시지를 넘긴다. 받은 메시지 위에 또 IP와 관련된 데이터들을 씌운다.
>5. 생성된 패킷이 랜카드를 통해서 밖으로 나갈 때 이더넷 프레임으로 감싸져서 나감.
